# Getting Started with Docsible

Welcome! This guide will get you up and running with Docsible in 5 minutes.

## What is Docsible?

Docsible automatically generates comprehensive documentation for your Ansible roles,
playbooks, and collections. Think of it as "auto-doc" for Ansible.

## Installation

```bash
pip install docsible
```

## Quick Start (2 minutes)

### 1. Navigate to Your Role

```bash
cd path/to/your/ansible-role
```

### 2. Generate Documentation

```bash
docsible role --role .
```

That's it! You now have a `README.md` with:
- Task documentation
- Variable documentation
- Handler documentation
- Role metadata

### 3. Review Your Documentation

```bash
cat README.md
```

## Common Commands

| Command | What it does |
|---------|-------------|
| `docsible role --role .` | Document current role |
| `docsible role --role . --graph` | With task flow diagrams |
| `docsible role --role . --minimal` | Quick minimal docs |
| `docsible role --role . --analyze-only` | Analyze without generating |
| `docsible role --role . --complexity-report` | Show complexity metrics |
| `docsible guide troubleshooting` | Common issues & fixes |

## What's Next?

### Generate Visualizations
```bash
docsible role --role . --graph
```

### See All Options
```bash
docsible role --help-full
```

### Check Role Quality
```bash
docsible role --role . --show-info --recommendations-only
```

## Pro Tips

1. **Smart Defaults**: Docsible auto-adjusts based on role complexity
2. **Positive Framing**: Use `--neutral` if you prefer plain output
3. **CI/CD**: Great for GitHub Actions automated documentation
